PRIVACY

    INTRODUCTION

  • This sevice only exists in the first place because of violations of privacy. Our data is leaked, sold, redistributed and abused to our detriment and beyond our control. We are established as a free service in 2019 to help provide a way to know if your current wallet is compromised or still safe to hold your coins. Whilst we may no longer be able to control it once breached, we can at least detect that it has been leaked and what precautionary measures you can now take as a result.

    This page details how the privacy of personal data is handled within our plaatforms and what information is collected when you use the service.

    BREACH DATA

  • When a data breach is registered compromised keys undergo a SHA-256 and the outpu hashes are stored in the online system. No other data of any kind (login details, identity documents, names, phone numbers, etc) are stored on data load. For example, it will state that the private key appeared in a breach it will use the key as an input to SHA-256(Secure Hash Algorithm) and store the outcome Hash, but no information about which key had which corresponding hash value are stored.

    When data is loaded into the compromised private key service, only SHA-256 hashes of the password are stored. No identifying data about to which key the hash belongs nor to which user it belonged to is stored.

    WHEN YOU SEARCH FOR A PRIVATE KEY

  • The searched address is never stored anywhere. The private key entered by the user is processed in a SHA-256 and the hash produced is compared to the SHA-256 hashes of compromised keys.

    If the hash corresponding to the entered private key is matched to the stored hash of a compromised key, the user will be notified that the their wallet is not safe to use.

    HOW USER-PROVIDED DATA IS HANDELED

  • The only data ever explicitly requested from users is the private key. The information is never passed to any third parties under any circumstances.
